# Project: Movies Ratings Analysis (Rotten Tomatoes)

## About Rotten Tomatoes
The movie rating system has become an integral part of the film industry, providing audiences with a benchmark to gauge movie quality. Among the various platforms, **Rotten Tomatoes** is one of the most popular and influential. Established in 1998 by Senh Duong, Rotten Tomatoes aggregates reviews from professional critics and audiences alike.  

The platform’s **Tomatometer score** is calculated by tallying the number of positive reviews from top critics against the total reviews. Movies with a score of **60% or higher** are considered *“fresh”*, while those below 60% are labelled *“rotten”*. This binary system offers audiences a clear, concise way to assess whether a movie is worth watching.

## About the Dataset
This dataset contains **over 15,000 movies** reviewed by Rotten Tomatoes. Each entry includes detailed information about:  

- Critic and audience ratings  
- Movie title, release year, and genre  
- Directors, writers, and cast  
- Plot summaries and production details  
- Theatre and streaming release dates  

The dataset allows exploration of the relationship between **audience reception and critical evaluation**, as well as trends across genres, years, and countries.
> **Note:** The full dataset is too large to host on GitHub. A **sample of the dataset** is included (`movies_sample.csv`) to demonstrate structure and analysis. The full dataset can be downloaded from [Kaggle]([https://www.kaggle.com/datasets/your-dataset-link](https://www.kaggle.com/datasets/subhajournal/movie-rating)).

## Project Focus
This project focuses on:  

- Writing SQL queries to explore, summarise, and analyse movie ratings and performance  
- Performing aggregations, joins, subqueries, and conditional logic  
- Using Excel or Power BI to create charts, pivot tables, and dashboards
