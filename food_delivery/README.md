# Food Delivery Data Analysis

## Description
This project uses the **Food Delivery Dataset**, part of the SQL Practice Dataset Series, which simulates a food delivery platform similar to services like Uber Eats or Deliveroo. The dataset contains realistic multi-table relational data, including customers, restaurants, menu items, orders, and individual items purchased in each order.

I analysed the dataset to understand patterns in customer orders, restaurant performance, menu popularity, and delivery efficiency.

## Project Focus
- Writing **SQL queries** to explore, summarise, and analyse the data  
- Performing **multi-table joins, aggregations, filtering, and subqueries**  
- Calculating **revenue, delivery performance metrics, and customer behaviour insights**  
- Using **Excel / Power BI** to create **charts, pivot tables, and dashboards** to visualise trends and key metrics  
- Identifying insights such as **top-performing restaurants, most popular menu items, peak order times, and delivery efficiency**

## Dataset Tables
The dataset contains 5 relational tables representing:

- **Customers** – Customer details and demographics  
- **Restaurants** – Restaurant information and cuisine types  
- **Menu Items** – Menu offerings and prices  
- **Orders** – Order transactions including date, total amount, and delivery details  
- **Order Items** – Individual items purchased within each order  

## Skills Practised
- **SQL:** Multi-table joins, aggregations, filtering, subqueries, revenue calculations, analytical queries  
- **Excel:** Pivot tables, charts, and dashboards to visualise order volumes, revenue, delivery performance, and popular menu items  
- **Data Analysis:** Examining customer behaviour, restaurant performance, ordering trends, and delivery metrics
